The 7 workstreams

3 already implemented (uncommitted on branch) 4 remaining
DoneW1 · models

Pin serialVersionUID

Pinned the JVM-computed values (LotteryGame = 8661…644, LotteryDraw = -8462…061) so existing .ser files still deserialize and future changes are controlled.

DoneW2 · serializer

Atomic + durable .ser write

Write to a temp file → fsync → atomic rename. Readers see the old or new file, never a half-written one. Base dir now configurable (default tmp).

DoneW3 · readers

HTTP timeouts

10s connect / 30s read on CSV & PDF downloads; PdfFileReader static lines → instance field (thread-safe for parallel poll).

To buildW4 · polling

Resilience4j + parallel, failure-isolated poll

Retry (3×, backoff) + rate limiter (polite to texaslottery.com) per game; bounded parallel fetch; one game failing no longer aborts the rest; safe state lookup (no throw). Poll logs at error, not debug.

To buildW5 · cache

In-memory catalog cache

Spring Cache on the hot read/catalog methods, evicted on a PollCompletedEvent published after each poll. Repeat catalog calls become free; data stays fresh within a poll cycle.

To buildW6 · errors

Global exception handling

@RestControllerAdvice with a structured ApiError body; 404 for missing state/game, 5xx for real failures. Replaces null+400 swallow. Logger/`System.out` cleanup.

To buildW7 · tests

Real-code tests

Point the service at a @TempDir via the new configurable base dir so the catalog test exercises the actual production method. Add serializer round-trip + atomicity (no leftover .tmp) tests.

Backward-compatibility contract

Unchanged
  • All 6 endpoint paths & HTTP methods
  • Success (2xx) response bodies & JSON shapes
  • On-disk layout tmp/{STATE}/{GAME}.ser (default)
  • Existing .ser files keep loading (pinned UID)
Intentional change (error paths only)
  • Errors now return proper 404/5xx + JSON error body instead of 400 + null
  • A client that treated the old 400/null as "empty" sees a status change — only on failures

Risks & how they're handled

RiskMitigation
Pinned UID differs from old files' implicit UIDComputed via serialver against the current class, so it equals the implicit value → old files still load. Verified before pinning.

Cache serves stale catalogEvicted on every PollCompletedEvent; poll is the only writer. Max staleness = one poll cycle.
